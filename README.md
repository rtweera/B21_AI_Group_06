## Repository Governance & QA
To ensure code quality and project integrity, the `main` branch is protected by a GitHub Ruleset. The following rules apply to all team members, including the repository owner (no bypasses allowed).


| Rule | Requirement | Goal |
| :--- | :--- | :--- |
| **Pull Requests** | Required for all changes | Prevent direct pushes to production. |
| **Peer Review** | 1 Mandatory Approval | Ensure a second pair of eyes on every line of code. |
| **Block Force Pushes** | Enabled | Protect the git history from accidental deletion. |
| **Conversation Resolution** | All comments must be resolved | Ensure feedback is addressed before merging. |
| **Linear History** | Disabled | Maintain flexibility for merge commits. |
| **Admin Bypass** | **OFF** | Enforce quality standards equally across the whole team. |

### Pull Request Workflow
1. Create a feature branch from `main`.
2. Push your changes and open a **Pull Request**.
3. Request a review from at least one teammate.
4. Resolve all review comments before merging.
